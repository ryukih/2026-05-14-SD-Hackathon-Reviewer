**評価日**: 2026-05-11

## 概要

AGI（汎用人工知能）が社会実装された近未来の労働市場をテーマにした、Web ベースのキャリアシミュレーター。シード連動型乱数により職種・業界・年齢・AI スキル等の属性を持つ 100 人の仮想プロフィールを自動生成し、「なにわ男子」風アイドルスキルを持つ特殊ペルソナも混在する。各個人は定型業務性・身体性・コミュニケーション能力等のパラメータから「AGI 適応力スコア」と「代替リスクスコア」を算出され、`resilient` ／ `transforming` ／ `vulnerable` の三類型に分類される。ユーザーは AGI 進展度（低・中・高）スライダーを操作することで、リアルタイムにスコア分布の変化を観察できる。個別プロフィールを選択すると NVIDIA NIM API（Llama 3.1）が「会社側の評価」「本人の内心・やりたいこと・今後のアクション・発言」を一括ナレーション生成し、さらに 100 人分の分布データを集計した労働市場全体の生存傾向レポートも自動生成する。Next.js（App Router）＋ TypeScript ＋ Tailwind ＋ Recharts 構成で、クライアントサイド乱数生成と非同期 LLM 呼び出し（キャッシュ付き）の組み合わせにより、軽量かつインタラクティブな UI を実現している。

## 評価詳細

### A. 創発設計 — 最終: 2.0/10

#### 評価
本作には**シミュレーションループ、エージェント間相互作用、メモリ、時間発展のいずれも存在しない**。100人のプロフィールは決定論的スコアリングで `resilient`/`transforming`/`vulnerable` に分類された後、ユーザーがクリックした個人について一度だけ LLM が「内心・発言・会社の判定」を**ナレーション**するのみであり、エージェントは世界を観測したり他者と通信したりしない。世界ルール設計精度は中程度（スコア式・AGIレベル乗数あり）だが、行動の自由度は皆無で、LLM はテキスト整形器として機能している。設計上、集団的創発が起こる余地が構造的に無い。

#### 根拠
- `src/app/api/agent/route.ts:31-53` — プロンプトは「以下の6点を出力してください ①現在のステータス…②会社の評価…③本来やりたいこと…④これからのアクション…⑤思考…⑥発言…」と出力項目を完全に指示しており、エージェントに行動選択の自由は無い。
- `src/app/api/agent/route.ts:55-60` — LLM 呼び出しは単発の chat completion、メッセージ履歴やメモリは無い。
- `src/components/ProfileDetail.tsx:60-86` — プロフィールが選択された瞬間に POST する一発呼び出し。返却データはセッション内 `agentCache` に格納されるが、エージェントの「記憶」として再利用される設計は無い。
- `src/lib/scoring.ts:17-58` — 全ての分類は決定論的な加重和で計算され、LLM の出力は分類に影響しない（一方向の従属関係）。
- README.md — エージェント同士の対話・観測・行動選択に関する記述は皆無。「LLMがリアルタイムに生成してロールプレイを行います」とあるが、ロールプレイ＝ナレーション生成であり、世界に作用しない。

### B. 世界設定 — 最終: 6.5/10

#### 評価
「AGI 導入下の労働市場における 100 人のキャリア帰結」というテーマは**社会的・ビジネス的関連性が高く**、雇用・自動化・スキル再構築という現代的論点を直接扱っている。21 職種 × 11 業界、7 つの個人スキル軸、AGI 進展度（低/中/高）の 3 段階トグルなど世界の構成要素は丁寧に設計されており、テーマフォーカスは強い。一方で「世界」は静的な人口分布の可視化に留まり、時間発展や相互作用が無いため**マルチエージェントシミュレーションとしての世界感は弱い**。題材自体の独自性は中程度（「AI による職の代替」可視化系で散見される）だが、アイドル風キャラを混ぜるなどのユニークな工夫はある。

#### 根拠
- `src/lib/config.ts:26-216` — 21 職種それぞれに `baseRoutineLevel`/`basePhysicality`/`baseCertificationDependence`/`hasHumanElement` を割り当てており、業界マッピングも含めて世界設計が丁寧。
- `src/lib/config.ts:248-252` — `AGI_LEVEL_MULTIPLIERS` により世界レベルの圧力（リスク倍率と適応力減衰）が定義されている。
- `src/lib/generateProfiles.ts:96-129` — 「道枝駿佑 (風)」「大橋和也 (風)」など固有ペルソナを初期挿入する遊び心ある世界観。
- README.md — 「AGI（汎用人工知能）が導入された世界における、100人の仮想プロフィールのキャリアシミュレーター」「どんなスキルの人が生き残り、どんな人がレイオフの危機に瀕するのか」 — テーマは明確だが、世界が時間発展しない静的可視化である点が世界設定の深さを制限。

### D. 技術実装 — 最終: 6.0/10

#### LLM利用
NVIDIA NIM API 経由で Llama 3.1 70B を OpenAI SDK 互換クライアントで呼び出す構成は妥当。プロンプトで JSON スキーマ指定し、出力に含まれる ```json``` フェンスを正規表現で剥がす後処理 (`route.ts:65`) もあり、`JSON.parse` 失敗時のフォールバックも実装済み。`temperature` (0.7/0.5)、`max_tokens` (800/1000) も明示的に設定。API キー未設定時のモックモードは UX 上の良い配慮。一方、構造化出力モード（response_format）や function calling は使用していない。

#### メモリ設計
**永続的・意味的なメモリ機構は存在しない**。`ProfileDetail.tsx:39` の `agentCache` はブラウザセッション内でプロフィール ID をキーに LLM 応答を保存するキャッシュに過ぎず、エージェントの履歴・学習・状態更新には使われない。マルチエージェントシミュレーションとして要求されるメモリ設計は実質的に欠如。

#### コード品質・シミュレーション正確性
TypeScript strict 環境、コンポーネント分離、型定義 (`types.ts`) の明示、命名の一貫性はいずれも良好。シード再現可能な乱数 (`generateProfiles.ts:15-21`)、`useMemo` による派生計算の整理 (`page.tsx:50-86`) も適切。ただし「シミュレーション」と呼べるステップ実行は存在せず、Recharts による静的可視化＋クリック契機の単発 LLM 呼び出しに留まる。同期性・レース条件等の検証対象自体が薄い。

#### 根拠
- `src/app/api/agent/route.ts:55-83` — chat completion 呼び出し、JSON 後処理、try/catch によるフォールバック。
- `src/app/api/agent/route.ts:18-29` および `src/app/api/trends/route.ts:17-22` — API キー未設定時のモック応答実装。
- `src/components/ProfileDetail.tsx:39` — `const agentCache: Record<number, AgentData> = {};` — モジュールスコープの単純な揮発キャッシュ。
- `src/lib/scoring.ts:131-151` — `scoreProfiles` は純粋関数として実装されており再現性が高い。
- `src/lib/types.ts:1-119` — 全ドメイン型を 1 ファイルに集約しており可読性が高い。
- `package.json:11-19` — Next.js 16、React 19、TypeScript 5 と最新スタック。

## 総評

### 優れている点
- **題材の社会的関連性が高い**: AGI 時代の雇用問題というタイムリーで重要なテーマを、21 職種・11 業界・複数スキル軸で多面的に表現している。
- **UI/UX とフロントエンド設計の完成度**: Recharts による散布図、AGI レベルスライダー、ProfileDetail のタブ切替、モーダルレポートなど、ハッカソンモックとして高水準。
- **コードの構造化と型安全性**: `lib/`（config・scoring・generateProfiles・types）/`components/`/`api/` の関心分離が明確で、新職種・新パラメータの追加コストが低い。
- **API キー未設定時のモック対応**: 評価者やデモ環境での動作確認を容易にする実務的配慮。

### 改善点・提言
- **マルチエージェントシミュレーションとしての要件不足**: シミュレーションループ、エージェント間相互作用、メモリ、時間発展のいずれも欠落しており、現状は「決定論的スコアリング ＋ 個別 LLM ナレーション」のレイヤード可視化に近い。例えば「他者の決定（離職・転職）を観測して自分の行動を更新する」ような相互作用ループの導入で創発設計スコアは大きく改善する。
- **LLM への行動自由度の付与**: 現状は出力項目（①〜⑥）を完全指定するナレーションプロンプトであり、エージェントは行動を選択していない。「次のターンで何をするか」を LLM に決定させ、世界状態を更新するアーキテクチャに転換すれば、創発の余地が生まれる。
- **メモリ設計の導入**: 現在の `agentCache` はキャッシュに過ぎない。エージェントごとに過去の意思決定・経験を蓄積し、次回プロンプトに含める構造にすると、ペルソナの一貫性と時間発展が両立できる。
- **将来構想の明文化**: README に「次ステップ：エージェント間で求人情報・転職経験を共有」「業界別 AGI 波及シミュレーション」など具体的展望を記載すると発展性評価が向上する。

### 一言コメント
ハッカソンモックとしてのフロントエンド完成度とテーマ性は高いが、「マルチエージェントシミュレーション」という評価軸では構造的に不足が大きく、現状は LLM 駆動の動的ペルソナ可視化ダッシュボードと位置付けるのが正確である。
