## 概要

寿命のある世界（A）と不老不死の世界（B）を並行して走らせる、A/B 比較型の LLM マルチエージェント・シミュレーション。各世界では 12 人の AI エージェントが 100 ステップにわたって生活し、A 世界ではエージェントが age と lifespan を持ち、死亡時に LLM が last_words（遺言）を生成して history_book.json に蓄積される。死亡したエージェントは generation+1 の新世代として再生成され、世代を跨いだ文化継承を観察可能にする。エージェントには Ambitious / Hedonist / Reflective / Anxious の 4 類型の性格システムが付与される。不老不死の B 世界では、死の不在がもたらす「無限の存在への不安」など、エージェント自発の実存的発話が観察される。テーマは「死は文化を生むのか／不老不死は祝福か牢獄か」という哲学的・人類学的問いで、ベースは兵頭氏作の「2D LLM Multi-places Simulation on Fire」、実行環境は Python 3.9 と Ollama + llama3.2 によるローカル運用である。

## 総評

### 優れている点
- 「死と文化」「不老不死と実存的不安」という哲学的・人類学的テーマを、A/B比較実験という分析設計に落とし込んだ世界設定の独自性
- `history_book.json` による世代間の永続的共有メモリと、generation+1 による世代継承メカニズムの組み合わせ
- 死亡時のLLMによる `last_words` 生成という、エージェント個性を物語化する仕掛け
- Ollamaによる完全ローカル実行で、再現性とコスト面で合理的な技術選択
- 不老不死世界における「無限の存在への不安」のような、設計者が意図していない可能性のある創発的発話の報告

### 改善点・提言
- ソースコードがリポジトリに含まれていないため、ハッカソンルールに従った差分（追加機能分）のみでもコード提示があれば、設計判断（プロンプト構造・性格類型の組み込み方・メモリ参照方法）を検証できる
- 将来展望・拡張ロードマップがREADMEにない。たとえば「5世代以上の長期実行」「性格類型の交配・進化」「他文明的テーマ（宗教生成、世代間紛争）への適用」など、本研究の延長線上で価値を生む方向性を明示するとC評価が大きく改善する
- 「3世代を経て文化が継承された」という観察結果について、何が継承されたか（特定の語彙、価値観、行動パターン）の定量・定性データを提示すると、研究としての説得力が増す
- A/B比較を支える指標（語彙多様性、感情極性、自己言及頻度など）の提示があると分析的価値が高まる
